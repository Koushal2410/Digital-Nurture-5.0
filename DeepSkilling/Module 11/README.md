# Module 11 - DevOps and CI/CD

This module covers DevOps principles, Continuous Integration (CI), and Continuous Delivery/Deployment (CD) automation practices.

---

## Key Learning Areas

### 1. DevOps Lifecycle & Practices
- **DevOps Definition**: Unifying development and operations teams to enable rapid, high-quality software releases.
- **Collaborative Workflows**: Infrastructure as Code (IaC), configuration management, automated testing, and release planning.

### 2. Continuous Integration & Continuous Delivery (CI/CD)
- **Continuous Integration (CI)**: Automatically building and running automated tests whenever code is merged to primary repository branches.
- **Continuous Delivery (CD)**: Automating the release process so changes can be deployed to staging or production environments reliably at any time.

### 3. CI/CD Tools & Platforms
- **GitHub Actions**: Creating YAML workflows (`.github/workflows/`) triggered by push or pull-request events.
- **Azure Pipelines**: Orchestrating enterprise-scale build and release pipelines.
- **Jenkins / GitLab CI**: Managing custom self-hosted pipelines.
