# Module 9 - Application Debugging

This module covers the core concepts and techniques for debugging frontend React/JavaScript applications using industry-standard browser developer tools and code editors.

---

## Key Learning Areas

### 1. Browser Developer Tools (Chrome DevTools)
- **DOM Inspection**: Modifying elements, testing layouts, and inspecting properties live in the browser.
- **Sources Panel**: Navigating files, setting breakpoints (line, conditional, and event listener breakpoints).
- **Console Panel**: Logging errors, examining stack traces, and executing javascript expressions in execution context.
- **Network Tab**: Inspecting fetch/XHR API requests, response payloads, headers, and checking latency/status codes.

### 2. React Developer Tools
- **Components Tab**: Inspecting React component trees, viewing and editing state and props live.
- **Profiler Tab**: Recording performance, diagnosing re-render bottlenecks, and checking component render timings.

### 3. Visual Studio Code Debugger
- Integrating VS Code with Chrome/Edge.
- Writing launch configurations (`launch.json`) to debug React applications directly from the workspace.
- Managing watches, local variables, call stacks, and stepping (Step Into, Step Over, Step Out).
